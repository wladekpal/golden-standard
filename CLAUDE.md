# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Research codebase for the paper "Is Temporal Difference Learning the Gold Standard for Stitching in RL?" — investigating whether TD methods truly outperform Monte Carlo methods at stitching experience fragments in reinforcement learning. Built entirely in JAX with a custom grid-world Box Moving environment.

## Common Commands

```bash
# Install dependencies (uses uv, not pip)
uv sync

# Run training with default CRL agent
uv run src/train.py env:box_moving --exp.name test

# Run training without wandb logging
uv run src/train.py env:box_moving --exp.name test --exp.mode disabled

# List all hyperparameters
uv run src/train.py --help

# Run tests (from repo root)
uv run pytest src/envs/block_moving/tests.py -v
uv run pytest src/envs/tests_wrapper.py -v
uv run pytest src/tests.py -v

# Lint (ruff; excludes src/impls/ and notebooks/)
uv run ruff check src/
uv run ruff format src/

# Collect expert offline dataset
uv run scripts/gather_expert_dataset.py --output-path data/out.npy --num-trajectories 100 --grid-size 6 --number-of-boxes-min 6 --number-of-boxes-max 6 --number-of-moving-boxes-max 6
```

## Architecture

### Training Pipeline (`src/train.py`)
Entry point. Uses `tyro` for CLI config parsing with subcommands (e.g., `env:box_moving`). The training loop is fully JIT-compiled via `jax.lax.scan`:
- `train_epoch` -> `train_interval` -> `update_step` (all wrapped in `@jax.jit`)
- Data collection happens inline: `collect_data` vmaps environment rollouts, then inserts into a replay buffer (`src/rb.py`)
- Evaluation runs separate rollouts and logs metrics + GIFs to wandb

### Configuration (`src/config.py`)
Three config groups parsed by tyro, controlled via CLI prefixes:
- `exp.*` — experiment settings (wandb, seeds, replay buffer, epochs, filtering)
- `env.*` — environment config (`BoxMovingConfig` in `src/envs/block_moving/env_types.py`)
- `agent.*` — algorithm hyperparameters (defined as `ml_collections.FrozenConfigDict` in `src/impls/agents/__init__.py`)

### Environment (`src/envs/block_moving/`)
A JAX-native grid-world where an agent moves boxes to targets. Key files:
- `block_moving_env.py` — core `BoxMovingEnv` with `reset`/`step` (all JAX-traceable)
- `env_types.py` — `BoxMovingState`, `TimeStep`, `GridStatesEnum` (12 cell states encoding agent/box/target combinations), `BoxMovingConfig`
- `generators.py` — level generators (`default` = random placement, `variable` = corner-based with optional `generator_special` diagonal mode)
- `wrappers.py` — `AutoResetWrapper`, `QuarterFilter` (training-time spatial filtering)
- `input_features.py` — grid encoding strategies (`raw_flat`, `normalized_flat`, `one_hot_flat`)

Actions: UP(0), DOWN(1), LEFT(2), RIGHT(3), PICK_UP(4), PUT_DOWN(5)

### Agents (`src/impls/agents/`)
Goal-conditioned RL agents without policy networks — actions sampled from Q-function via softmax. Agent creation is dispatched by `agent_name` string in `create_agent()`. The paper-relevant agents (used with `_search` suffix) are:
- `crl_search` — Contrastive RL (MC-style, no rewards)
- `clearn_search` — C-Learning (TD-style, no rewards)
- `gciql_search` — Goal-Conditioned IQL (TD and MC variants, with rewards)
- `gcdqn` — Goal-Conditioned DQN (TD and MC variants, with rewards)

TD vs MC behavior is toggled via `--agent.is_td` / `--agent.use_discounted_mc_rewards` flags.

### Optimal Planner (`src/optimal_q.py`)
Computes optimal solutions using Hungarian algorithm (via `ott-jax`) for box-target assignment + Manhattan-distance action sequences. Two versions:
- `solve_state()` — Python-loop solver for single states
- `solve_state_padded()` / `solve_state_vmapped()` — JIT-compatible padded version for batched parallel collection

### Replay Buffer (`src/rb.py`)
`TrajectoryUniformSamplingQueue` — stores full trajectories and samples with geometric future-goal relabeling via `flatten_batch`. Goals can be future states (geometric sampling) or random (rolled from other trajectories), controlled by `use_future_and_random_goals`.

## Key Patterns

- All environment logic must be JAX-traceable (no Python control flow on traced values — use `jax.lax.cond`/`jax.lax.scan` instead)
- Scripts that import from `src/` manipulate `sys.path` to add both the repo root and `src/` directory
- Agent checkpoints are pickled dicts with `flax.serialization` state + config, saved to `runs/` or `--exp.save_dir`
- Experiments directory contains self-contained experiment snapshots (config + results + README), generated by SLURM batch jobs
- `ruff` is configured with `line-length = 120` and excludes `src/impls/` and `notebooks/`
